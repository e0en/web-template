웹 디자인 템플릿

개인적으로 사용할 웹 사이트에 일관적으로 적용할 마크업, 스타일을 정의한다.

개발 동기

본업이 웹 개발은 아니지만 취미 혹은 필요로 인해 여러개의 웹 페이지를 만들게 된다. 매 작업마다 별개의 스타일시트를 작성하니 중복 작업도 많고 결과물도 만족스럽지 않았다. 이 문제를 해결하려고 bootstrap, bulma 등의 웹 프레임워크를 써 보았지만 다음과 같은 점이 아쉬웠다.

  • 프레임워크에서만 사용하는 HTML tag 클래스 강제
  • 원하지 않는 스타일 정의가 너무 많아 무거움
  • 네비게이션, 폼에 전용 마크업 강제
  • css variable 대신 scss, less와 같은 CSS 전처리기 사용

위의 내용이 객관적인 단점이라고 하기는 어려우나, 나의 작업 방식과 취향에는 맞지 않았다. 웹 디자인에 대한 단편적 지식들을 이 페이지에 코드화하여 향후 작업에 활용하고 싶다.

참고자료

다음과 같은 프로젝트에서 영향을 받았다.

  • MDN의 HTML5 semantic markup 가이드
  • semrush의 semantic markup 가이드
  • 신현석 님의 HTML5 마크업
  • 98.css

개발 목표

위에서 언급한 것처럼 개인 프로젝트에 폭넓게 사용할 수 있는 템플릿을 만드는 것이 목적이다. 이상적으로는 디자인 시스템 같은 것을 만들어야 하겠지만, 디자인 문외한이 쉽게 할 수 있는 일은 아닌 것 같기에 그나마 현실적으로 목표를 조정했다.

이 작업에서 만족해야 할 내용을 중요도 순으로 나열해 보았다.

  1. Mobile first. 모바일 스타일을 먼저 정의하고 데스크탑 스타일을 추가로 정의한다
  2. 전용 마크업을 최소화하고 최대한 HTML tag 이름으로 구분한다
  3. 한글 텍스트에서도 보기 좋은 글꼴을 디폴트로 선정. Noto Sans KR을 쓸 것이다
  4. javascript 최소화. 내비게이션/폼에도 되도록이면 쓰지 않을 예정이다
  5. CSS variable로 변수를 정의해, 개별 프로젝트에서 커스텀을 하더라도 :root 요소만 수정해도 되도록 한다
  6. 제대로 마크업하기 힘든 폼 같은 요소는 스타일뿐 아니라 마크업 가이드도 신경써서 작성해 재활용하기 좋게 작업한다
  7. 다크 모드 지원

네비게이션

어느 화면에서도 쓰기 좋은 메뉴를 만드는 것은 어렵다.

  • 로고 넣을 자리는 마련해두고 싶다.
  • 화면의 세로 폭은 제한된 자원이기 때문에 최대한 절약하고 싶다.
  • 모바일 화면에서 한물 간 햄버거 버튼 없이 메뉴를 잘 표시하고 싶다.
  • 내가 쇼핑몰처럼 넓고 깊은 카테고리 네비게이션을 만들 일은 당분간 없을 것 같다.
  • 검색은 웬만한 웹사이트에는 반드시 필요하다.
  • 모바일 화면에서는 화면 상단에 손을 뻗을 필요 없게 하고 싶다.
  • 데스크탑에서는 화면 좌측을 활용해볼까?

폼 입력 양식

다른 문서 요소와는 달리 좋은 폼을 만들려면 스타일시트뿐만 아니라 마크업도 잘 구성해야 한다. 따라서 다양한 마크업에 대응할 수 있도록 스타일시트를 유연하게 정의하고, 이 문서에서는 마크업 예제를 제시해서 다른 프로젝트에서 사용할 수 있도록 하자.



의미가 아닌 형태를 정의하는 br 태그를 썩 좋아하지는 않지만 label 태그와 함께 쓸 때에는 대안이 없다. label 태그와 input 태그의 상대 위치가 상황에 따라 달라야 하기 때문이다.
